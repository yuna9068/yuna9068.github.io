MySQL 比較運算子/邏輯運算子/SQL特定運算子

2017-08-22
MySQL, 臺北科大Java

運算式中的運算元資料形態要相同

運算子優先順序

  1. 算數運算子(、/ 優先於 +、-； 與/ (或 +與- )同時出現時，由左至右)
  2. 串連運算子
  3. 比較運算子
  4. IS NULL, LIKE, IN
  5. BETWEEN
  6. NOT
  7. AND
  8. OR
    使用()改變運算順序，也可讓運算式意義更清楚

比較運算子

比較運算子使用在兩個資料項的比較大小
　　= 等於　　>大於　　>=大於或等於　　<小於　　<=小於或等於　　<>不等於
　　不能寫 100<=X<=200 ，要寫成 X>=100 AND X<=200

  • 數值資料：值
  • 字串資料：內碼(預設)英文→ASCII(第一碼0)；中文→BIG5(第一碼1)
    　　　　　中文>英文小寫>英文大寫>阿拉伯數字
    　　　　　比第一個字即可，若第一個字相同則比下一個字
  • 日期時間資料：世紀、年、月、日、時、分、秒

邏輯運算子

若有一個以上的條件運算必須使用邏輯運算子結合成一個運算結果
　　AND　　OR　　NOT

SQL特定運算子

  • BETWEEN：連續區間值的查詢，值由小→大，不可以大→小
    　　　　　　可使用在數值、日期、字元
    　　　　　　WHERE sal >= 800 AND sal <= 1000 → WHERE BETWEEN 800 AND 1000
  • IN：列舉值的查詢
    　　可使用在數值、日期、字元
    　　WHERE sal = 800 OR sal = 900 OR sal = 1000 → WHERE sal IN(800, 900, 1000)
  • LIKE：萬用字元的查詢
    　　　%任意字元　　_一個字元
    　　　WHERE name LIKE ‘A%’→A開頭
    　　　WHERE name LIKE ‘%A’→A結尾
    　　　WHERE name LIKE ‘%A%’→含有A
    　　　WHERE name LIKE ‘__A%’→第三個字是A

　　　若要搜尋3% → LIKE ESCAPE →

　　　ESCAPE ‘&’跳脫符號，取消符號後方字元的特殊性，符號自訂，例：&、\、*

  • ISNULL：空值(NULL)的查詢
    　　　　 WHERE sal IS NULL(不可以寫成sal=NULL)

資料來源：李志賢老師上課講義
